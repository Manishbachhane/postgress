SQL commands ko hamesha ke liye yaad rakhne ka sabse best tarika hai isko ek Real-Life Analogy (kisi aam zindagi ke kaam) se jod dena.Hum database ko ek "Naya Ghar (House)" banane aur usme rehne ke example se samjhenge. Is kahani ko ek baar padhein, aap ise kabhi nahi bhulenge.

1. DDL (Data Definition Language) - "Ghar ka Structure Banana"Concept: DDL ka kaam sirf ghar ki building, deewarein aur nakshe (blueprint) par hota hai. Iska andar rakhe saaman (data) se koi lena-dena nahi hai. Ye database ka structure tay karta hai.Commands:CREATE: Khali zameen par naya ghar ya naya kamra banana (Nayi Table create karna).ALTER: Bane hue ghar me ek nayi khidki ya darwaza nikalna (Table ke structure me naya column add karna ya change karna).DROP: Buldozer lagakar pura ghar hi tod dena, na ghar bacha na saaman (Table ko root se completely delete kar dena).TRUNCATE: Ghar ke andar ka saara saaman ek sath jhaadu maar ke bahar nikal dena, lekin ghar ki deewarein (structure) wahi rahengi (Table ko pura khali kar dena).

2. DML (Data Manipulation Language) - "Ghar ka Saaman aur Log"Concept: Ghar ban chuka hai (DDL ne bana diya). Ab is ghar ke andar saaman rakhna, hatana ya badalna DML ka kaam hai. Ye sirf Data ke sath khelta hai.Commands:INSERT: Ghar me naya sofa ya naya TV lekar aana (Table me naya Data/Row daalna).UPDATE: Purane dabe wale TV ko hata kar waha Smart TV laga dena (Pehle se maujud Data ko modify karna).DELETE: Kisi ek tute hue kursi ko chun kar ghar se bahar nikal dena (Table me se kisi specific Data/Row ko delete karna)

.3. DCL (Data Control Language) - "Ghar ki Security aur Chaabi"Concept: Ab aapka ghar ban gaya aur saaman bhi aa gaya. Lekin ghar me ghusne kisko dena hai aur kise nahi? DCL database ka "Security Guard" hai.Commands:GRANT: Apne dost ko apne ghar ki chaabi de dena taki wo andar aa sake (Kisi user ko database use karne ki Permission dena).REVOKE: Dost se ladaai hone par usse chaabi wapas chheen lena (Di gayi Permission ko wapas le lena).

4. TCL (Transaction Control Language) - "Ghar ki Setting ka Undo/Save Button"Concept: Jab aap DML (Ghar me saaman jama rahe ho) ka use karte hain, toh TCL aapko sahuliyat deta hai ki kaam ko "Save" karna hai ya galti hone par "Undo" karna hai.Commands:COMMIT: Aapne saaman jamaya, aapko pasand aaya aur aapne setting final karke lock kardi (Changes ko permanently Save kar dena).ROLLBACK: Aapne neele rang ka parda lagaya, accha nahi laga, toh aapne turant usko hata kar wapas pehle jaisa kar diya (Galti hone par pichle state me Undo kar dena).SAVEPOINT: Saaman jamate waqt ek 'Check-point' bana lena. Jaise, "Yaha tak ki setting theek hai, isko save point bana lo, iske aage kuch gadbad hui toh pura shuru se nahi, bas yahi tak wapas aayenge" (Transaction me ek bookmark set karna).

Scenario 1: RESTRICT (Safe but Strict)

->Is table mein agar koi order pada hai, toh tum uss user ko delete nahi kar paoge.
Database error phenk dega.

->ON DELETE... ka code nahi likhte ho, toh default behavior RESTRICT hi hota hai.

CREATE TABLE orders_restrict (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    total_amount DECIMAL(10, 2),
    
    -- Yahan hum column define kar rahe hain
    user_id INT, 
    
    -- Yeh hai Asli Jadoo (The String)
    FOREIGN KEY (user_id) REFERENCES users(user_id) 
    ON DELETE RESTRICT ---->yha se restrict ho rha he
    ON UPDATE RESTRICT ---->yha se restrict ho rha he
);


Scenario 2: CASCADE (The Domino Effect)
->Agar Amazon jaisi app banani hai, jahan user ke delete hote hi uski saari search history ya cart items apne aap delete ho jayein, toh hum CASCADE use karenge.

CREATE TABLE orders_cascade (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    product_name VARCHAR(200),
    
    user_id INT, 
    
    -- Ek ko udaoge, toh saare orders udd jayenge
    FOREIGN KEY (user_id) REFERENCES users(user_id) 
    ON DELETE CASCADE 
    ON UPDATE CASCADE
);


Scenario 3: SET NULL (The Orphan Data)
->Isme agar user delete hua, toh orders table se row delete nahi hogi. Bas user_id ki jagah khali (NULL) ho jayegi, taaki account delete hone ke baad bhi tumhara total profit calculate hota rahe.

CREATE TABLE orders_setnull (
    order_id INT PRIMARY KEY AUTO_INCREMENT,
    bill_amount DECIMAL(10, 2),
    
    -- DHYAN DEIN: Yahan galti se bhi 'NOT NULL' mat likhna
    user_id INT, 
    
    -- User gaya, par data bacha liya
    FOREIGN KEY (user_id) REFERENCES users(user_id) 
    ON DELETE SET NULL ->>>yha se hoga 
);
