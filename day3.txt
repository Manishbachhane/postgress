//Scalar Subquery ka Golden Rule: "The Rule of ONE"

-->Rule: Scalar subquery ko strictly sirf ek value (1 row aur 1 column) hi return karni chahiye.

-->Operator Limitation: Iske bahar aap hamesha single-value operators ka hi use karenge jaise =, >, <, >=, <=, !=.

-->Safety Hack: Agar inner query galti se multiple rows de sakti hai, toh syntax error aa jayega. Isse bachne ke liye inner query mein hamesha Aggregate function (MAX(), MIN(), AVG()) ya LIMIT 1 ka use karein.

-->Galat: WHERE salary = (SELECT salary FROM employees) (Error dega kyunki list aayegi)

-->Sahi: WHERE salary = (SELECT MAX(salary) FROM employees) (Hamesha ek hi value aayegi)

SELECT emp_name 
FROM employees 
WHERE salary = (SELECT MAX(salary) FROM employees);
SELECT emp_name 
FROM employees 
WHERE salary > (SELECT AVG(salary) FROM employees);
SELECT emp_name 
FROM employees 
WHERE salary < (SELECT MIN(salary) FROM employees);
SELECT emp_name 
FROM employees 
WHERE salary >= (SELECT MIN(salary) FROM employees WHERE dept_id = 1);
SELECT emp_name 
FROM employees 
WHERE salary != (SELECT AVG(salary) FROM employees);

-->2. Nested (Multi-row) Subquery ka Golden Rule: "Never use Equals (=)"

-->Rule: Agar aapki inner query ek list return karne wali hai (multi-row), toh kabhi bhi = (equal to) operator ka use na karein.

-->Operator Rule: List wali subqueries ke bahar hamesha IN, NOT IN, ANY, ya ALL jaise multi-row operators ka use karna compulsory hai.

-->Column Matching: Jo column aap outer query ke WHERE mein compare kar rahe hain, inner query ke SELECT mein bhi exactly wahi same column hona chahiye.

-->Galat: WHERE dept_id = (SELECT dept_id FROM departments) (Multiple values ke sath = kaam nahi karta)

-->Sahi: WHERE dept_id IN (SELECT dept_id FROM departments) (Perfect)

-->3. Correlated Subquery ka Golden Rule: "Alias (Nickname) is Mandatory"

-->Rule: Correlated subqueries aapas mein judi hoti hain. Isliye database ko confusion se bachane ke liye outer table aur inner table dono ko Alias (nickname) dena 100% zaroori hai.

-->The Connecting Link: Inner query ke WHERE clause mein aapko hamesha inner_alias.column = outer_alias.column likhkar un dono ke beech ek link (pul) banana padega. Is link ke bina aapki query correlated nahi banegi aur galat results degi.

-->Galat: WHERE salary > (SELECT AVG(salary) FROM employees WHERE dept_id = dept_id) (Database confuse ho jayega ki kiska dept_id)

-->Sahi: FROM employees AS outer_table ... WHERE inner_table.dept_id = outer_table.dept_id (Crystal clear link)

SELECT p1.name 
FROM products p1
JOIN (SELECT category, AVG(price) as avg_price FROM products GROUP BY category) p2
ON p1.category = p2.category
WHERE p1.price > p2.avg_price;


1. Execution Flow (Kaise kaam chalta hai)
Outer Query apni pehli row uthati hai.

Outer Query us row ka data (jaise category = 'Electronics') Inner Query ko bhejti hai.

Inner Query us data ko use karke apna kaam (Average, Min, Max) karti hai.

Inner Query apna result Outer Query ko wapas return karti hai.

Outer Query check karti hai: "Kya meri condition match hui?"

Repeat: Yeh process table ki har ek row ke liye dobara hoti hai.

Step 1: Outer Query likho (Pehle main table socho)
Mujhe products chahiye.

SQL
SELECT name, price, category 
FROM products AS p1
Step 2: Comparison socho (Kya check karna hai?)
Mujhe price ko compare karna hai.

SQL
SELECT name, price, category 
FROM products AS p1
WHERE price < ( ... )
Step 3: Inner Query likho (Connection banao)
Ab mujhe wahi category ka average nikalna hai jo bahar wali row ki category hai.

SQL
SELECT AVG(price) 
FROM products AS p2
WHERE p2.category = p1.category -- Yahi "Magic Link" hai!
